Privacy, legal & security

Last updated 17 July 2026. FigureLoom is an open scientific illustration workspace maintained in the FigureLoom GitHub repository.

No personal email is published here. General questions, privacy requests and non-sensitive bug reports can be submitted through the project’s GitHub issue tracker. Never place passwords, access tokens, private health information or unpublished research data in a public issue.

Local and cloud data

The editor works locally without an account. Local projects, recovery copies and preferences remain in the browser unless the user exports or deliberately saves to the cloud. Signed-in accounts use Supabase for authentication, encrypted cloud projects, project membership, collaboration presence and encrypted comments.

Account information

When an account is created, FigureLoom processes the email address, authentication records, display name, selected profile symbol, project-access roles and timestamps required to operate the service. Passwords are handled by Supabase Auth and are not stored in the FigureLoom repository.

Project encryption

Editable cloud-project payloads and collaboration-comment bodies are encrypted in the browser before database storage. Project titles, ownership, roles, revision numbers and timestamps remain operational metadata. This is application-layer encryption rather than a zero-knowledge guarantee.

Third-party infrastructure

  • GitHub hosts the source repository and static website.
  • Supabase provides authentication, database and realtime collaboration infrastructure.
  • Google Gemini provides optional online AI when enabled by the user.
  • The domain registrar and DNS provider route figureloom.org to the website.

AI and personal API keys

FigureLoom sends a compact figure summary and the user’s prompt to Gemini only when the user asks for AI assistance. Embedded image bytes and raw SVG source are not included. A personal Gemini API key may be supplied for a request; it is not stored in the project or cloud gallery. Users should avoid sending confidential or unpublished research and should review Google’s terms for the API key being used.

Acceptable use and warranty

Do not use the service to violate law, distribute malicious material, impersonate another person or expose information without permission. Scientific and accessibility checks are assistive tools and do not replace professional review. The software is provided without a promise that it will be uninterrupted, error-free or suitable for a particular publication, clinical or regulatory purpose.

Open-source status

The source is publicly available on GitHub. A formal software licence should be reviewed in the repository before copying, modifying or redistributing the code; public source visibility alone does not create licence permission.

Contact and requests

Use the GitHub issue form for ordinary questions, data-access/deletion requests and non-sensitive reports. Sensitive security information should not be posted publicly; a private reporting route will be added before a broader public launch.
