# Interface update: 20 July 2026

This update focused on consistency, navigation, and avoiding regressions. It did not change the FigureLoom project format.

## Desktop interface

- Settings, Projects, Add, Loomy, Pro Tools, and the inspector now use the same compact interface scale.
- The Help button remains a proper circle and is protected from being squeezed by nearby actions.
- The collaboration profile control is compact instead of taking over the title bar.
- Science Library illustration cards remain large enough to show the artwork clearly.
- Projects → Current keeps reconnect, new-window, and close actions readable instead of crushing them into tiny boxes.
- The close **×** for an open project tab sits beside the project title. It closes the tab only and does not delete the saved project.
- The Pages, Hand, zoom, 100 percent, Format, and Navigation bar moves as one draggable control bar and keeps its existing collapse behavior.

## Phone interface

- **More → Help** opens the real FigureLoom Help center.
- The Help center links to the full manual, Start here, quick task guides, the visual guide, and the passive interface tour.
- The phone Help action is named Help rather than pointing at an obsolete Guide control.

## Passive interface guide

The passive guide now covers more of the interface.

Desktop coverage includes project tabs, Add, Illustrations, the movable canvas controls, pages and layers, the inspector, Pro Tools, Share, light and dark appearance, Help, autosave, recovery, and export.

Phone coverage includes the compact header, scrollable tabs, canvas, bottom dock, Tools, Pages, Edit, More, Add, Help, canvas navigation, and export.

The guide remains passive. It does not open panels, move or select objects, change project data, or scroll the project.

## Light and dark appearance

The affected project, collaboration, inspector, Help, and navigation surfaces use the shared FigureLoom theme variables in both light and dark appearance. The canvas and exported page keep their own project colors.

## Stability checks

A focused desktop and phone browser test now checks:

- project-tab close placement
- phone Help opening the real Help center
- passive-guide step counts and non-interference
- uncaught page errors
- `console.error` output during the tested flows

See [Troubleshooting and recovery](Troubleshooting-and-Recovery) if an old cached layout is still visible.
